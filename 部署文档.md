# 甘欣欣公司库存管理系统 — 产品打包与异地部署文档

本文档说明如何将本项目（前端静态页 + Node.js 后端 + MySQL 数据库）打包，并在**与开发环境不同的机器**上完成安装与上线。适用于 Windows Server / Linux 服务器场景。

---

## 一、系统架构说明

| 组成部分 | 说明 |
|---------|------|
| **前端** | `public/` 目录下的 HTML / CSS / JS，由 **Express 静态托管**，不单独部署 Nginx 站点也可访问 |
| **后端** | `server.js`，Node.js 进程，默认监听 **TCP 3000** |
| **数据库** | MySQL 5.7+ / 8.x，表结构见 `init.sql`；程序启动时会执行 `CREATE TABLE IF NOT EXISTS` 等初始化（异地库需网络可达且账号有建表权限） |

**访问关系（典型）：**

```text
用户浏览器 ──HTTP──► 应用服务器:3000 (Node 提供页面 + /api/*)
                           │
                           └──MySQL 协议──► 数据库服务器:3306（可同机或异地）
```

## 二、部署前准备清单

### 2.1 应用服务器（运行 Node 的机器）

- 操作系统：Windows Server 2016+ / Windows 10+ / Ubuntu 20.04+ / CentOS 7+ 等
- 安装 **Node.js LTS**（建议 18.x 或 20.x），并确认命令行可用：
  - `node -v`
  - `npm -v`
- 若使用 PowerShell 执行 `npm` 报错（执行策略），参考项目内说明或使用 `npm.cmd`
- 开放防火墙入站端口：**3000**（或您改为 80/443 后的端口）

### 2.2 数据库服务器（MySQL，可与应用同机或异地）

- 安装 **MySQL 5.7+ 或 8.0+**
- 创建业务库（示例库名 `dbms`，可按需改名）
- 创建专用账号并授权（**不要使用 root 长期跑业务**）
- 若数据库在**另一台机器**，需：
  - MySQL 配置 `bind-address` 允许远程（如 `0.0.0.0` 或内网 IP）
  - 云安全组 / 防火墙放行 **3306**（建议仅允许应用服务器内网 IP）
- 字符集建议：`utf8mb4`，排序规则 `utf8mb4_unicode_ci`（与中文、发票号等兼容）

### 2.3 网络与安全

- 生产环境强烈建议：**HTTPS**（反向代理终止 TLS）+ **强密码** + **JWT 密钥独立配置**（见下文「配置项」）
- 当前代码中 `JWT_SECRET`、数据库密码等为**硬编码示例**，上线前务必修改

## 三、产品打包流程（发布包制作）

### 3.1 打包应包含的文件（推荐）

在**开发机**项目根目录执行打包前，建议目录结构如下：

```text
products_manage_system/
├── server.js              # 后端入口（必带）
├── package.json           # 依赖声明（必带）
├── package-lock.json      # 锁定版本（强烈建议带上）
├── init.sql               # 建表脚本（异地手工建库/核对用，必带）
├── start-backend.bat      # Windows 一键启动（可选）
├── public/                # 前端全部静态资源（必带）
│   ├── index.html
│   ├── register.html
│   ├── *.html / *.js / style.css
│   └── ...
└── （不要打包整个 node_modules，见下）
```

### 3.2 不应直接拷贝的内容

- **`node_modules/`**：体积大、且可能与目标机 CPU/系统不兼容；应在**目标服务器**上执行 `npm ci` 或 `npm install --production` 重新安装。

### 3.3 推荐打包命令（开发机）

**Windows PowerShell 示例**（在项目根目录）：

```powershell
# 进入项目根目录
cd "D:\MyFiles\VibeCoding项目\products_manage_system"

# 可选：删除本地 node_modules 后再打 zip，减小误传体积（部署机仍会 npm install）
# Remove-Item -Recurse -Force node_modules -ErrorAction SilentlyContinue

Compress-Archive -Path server.js,package.json,package-lock.json,init.sql,start-backend.bat,public -DestinationPath ..\inventory-system-release.zip -Force
```

**Linux / macOS 示例：**

```bash
cd /path/to/products_manage_system
zip -r ../inventory-system-release.zip server.js package.json package-lock.json init.sql start-backend.bat public
```

### 3.4 版本与记录

建议在压缩包名或说明中带**版本号与日期**，例如：`inventory-system-v1.0.0-20260328.zip`，并附 `CHANGELOG` 或简短发布说明（修了哪些问题）。

## 四、数据库异地安装与初始化

### 4.1 在 MySQL 服务器上创建库与用户

以 MySQL 8 为例（**请替换密码与库名**）：

```sql
CREATE DATABASE IF NOT EXISTS dbms DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'fqc'@'%' IDENTIFIED BY 'YourStrongPasswordHere';
-- 若仅允许应用服务器 IP：
-- CREATE USER 'fqc'@'10.0.0.5' IDENTIFIED BY 'YourStrongPasswordHere';

GRANT ALL PRIVILEGES ON dbms.* TO 'fqc'@'%';
FLUSH PRIVILEGES;
```

说明：

- `'fqc'@'%'` 允许任意主机连接，生产环境建议改为**应用服务器内网 IP**。
- 若应用只建表不改库结构，可只授予 `CREATE, ALTER, SELECT, INSERT, UPDATE, DELETE` 等最小权限（需按贵司安全规范调整）。

### 4.2 导入 `init.sql`（可选但推荐）

在**已选好库**的前提下导入，用于与线上一致、或首次无网络时预建表：

```bash
mysql -h 数据库IP -u fqc -p dbms < init.sql
```

注意：应用首次启动也会执行 `CREATE TABLE IF NOT EXISTS`；若您希望**完全由 DBA 管控 DDL**，可关闭应用内自动建表（需改代码，本仓库默认**自动建表**）。

### 4.3 远程连接验证（在应用服务器上）

```bash
mysql -h <数据库服务器IP> -P 3306 -u fqc -p dbms -e "SELECT 1;"
```

能连通后再部署 Node。

## 五、应用服务器部署步骤

### 5.1 解压发布包

将 `inventory-system-release.zip` 解压到固定目录，例如：

- Windows：`D:\apps\inventory-system\`
- Linux：`/opt/inventory-system/`

### 5.2 安装依赖

```bash
cd /opt/inventory-system   # 或 Windows 对应路径
npm ci --omit=dev
# 若无 package-lock.json，则使用：
# npm install --production
```

### 5.3 修改数据库与端口配置（必做）

当前 `server.js` 中配置为：

```js
const PORT = 3000;
const dbConfig = {
  host: "127.0.0.1",
  user: "fqc",
  password: "Admin@123",
  database: "dbms",
};
const JWT_SECRET = "products_manage_system_secret_key";
```

**异地部署时请修改：**

| 配置项 | 说明 |
|--------|------|
| `dbConfig.host` | 改为**数据库服务器 IP 或域名**（若同机则 `127.0.0.1`） |
| `dbConfig.user` / `password` / `database` | 与 4.1 中创建的一致 |
| `JWT_SECRET` | 改为随机长字符串（生产必改，否则 token 易被伪造） |
| `PORT` | 若需 80 端口，可改为 `80`（Linux 需 root 或授权；更推荐 Nginx 反代） |

保存后重新启动进程。

### 5.4 启动服务

**前台调试（确认无报错）：**

```bash
node server.js
```

看到控制台输出 `Server running: http://localhost:3000` 且**无**「数据库初始化失败」即表示数据库连通、表结构初始化成功。

**Windows 可使用项目自带：**

- 双击 `start-backend.bat`（会尝试释放 3000 端口后启动）

**Linux 后台常驻（示例 systemd）：**

创建 `/etc/systemd/system/inventory-system.service`：

```ini
[Unit]
Description=Inventory System Node App
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/inventory-system
ExecStart=/usr/bin/node server.js
Restart=always
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable inventory-system
sudo systemctl start inventory-system
sudo systemctl status inventory-system
```

### 5.5 使用 Nginx 反向代理（推荐生产）

用户访问 `https://yourdomain.com`，由 Nginx 转发到 `127.0.0.1:3000`，并配置 SSL 证书。示例片段：

```nginx
server {
    listen 443 ssl;
    server_name yourdomain.com;
    ssl_certificate     /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 六、前端与访问说明

- 本系统**无独立前端构建步骤**；页面在 `public/`，由 `express.static("public")` 提供。
- 用户浏览器访问：`http://应用服务器IP:3000/`（或通过域名 + 反代）。
- 所有 API 为**同源** `/api/*`，无需单独配置 CORS 给静态页（若未来前后端域名分离，需再改 CORS）。

1. 打开登录页，标题显示为「甘欣欣公司库存管理系统」。

## 八、常见问题（FAQ）

### Q1：启动报「数据库初始化失败」

- 检查 `dbConfig.host`、端口、用户名密码、库名是否正确。
- 检查 MySQL 是否允许远程、`bind-address`、防火墙、安全组。
- 检查账号是否有 `CREATE TABLE` 权限（应用会建表）。

### Q2：浏览器一直提示「网络异常」

- 多为**后端未启动**或**端口被占用**导致请求失败；在服务器本机 `curl http://127.0.0.1:3000/` 测试。
- 若前端通过域名访问而 API 仍指向旧地址，需检查是否用了绝对 URL（当前项目为相对路径，一般无此问题）。

### Q3：MySQL 版本较老，不支持某些语法

- 本项目已针对部分版本做过 `ALTER TABLE` 兼容处理；若仍报错，请记录完整错误信息并对照 MySQL 版本文档升级或调整 SQL。

### Q4：如何把依赖打进一个可执行文件？

- 可使用 `pkg`、`nexe` 等将 Node 应用打成单文件，但需单独处理 `mysql2` 原生模块与静态目录 `public`，复杂度较高；**常规生产部署推荐本文的「Node + npm install + systemd/Nginx」方式**。
